# 伯俊ERP · 智能经营分析助手 — 部署方案

## 一、方案概述

为鞋服零售连锁客户提供基于伯俊ERP的智能经营分析系统，包含：
- 本地化部署（客户自购 Mac Mini）
- 伯俊ERP数据对接（REST API）
- 经营分析报表（Web端 + 企业微信）
- AI数据查询助手

## 二、系统架构

```
┌─ 客户浏览器 ───────────────────────┐
│  (企业微信/钉钉/浏览器打开报表)      │
└──────────┬─────────────────────────┘
           │
           ▼
┌─ 伯俊ERP 动作定义 ─────────────────┐
│  pc.navigate(url, 'ifr', '报表')   │
│  嵌入伯俊内部打开报表               │
└──────────┬─────────────────────────┘
           │
           ▼
┌─ 代理服务 (客户Mac Mini) ──────────┐
│  Node.js / Express                 │
│  端口: 3000                        │
│  功能: 签名转发 + 数据格式化 +      │
│        报表聚合 + 用户鉴权          │
└──────────┬─────────────────────────┘
           │ 加 AppKey/AppSecret 签名
           ▼
┌─ 伯俊ERP REST API ────────────────┐
│  http://8.130.178.145:90/...      │
│  按用户权限过滤数据                │
└────────────────────────────────────┘
```

## 三、部署步骤

### 3.1 客户准备
- [ ] 客户自购 Mac Mini（M系列芯片，8GB+内存）
- [ ] 伯俊ERP账号已分配好权限
- [ ] 客户自行选择并付费开通 AI 模型 API（DeepSeek/OpenAI等）

### 3.2 你来做

#### 第一步：Mac Mini 基础环境
```bash
# 安装 Node.js
brew install node

# 安装小龙虾 OpenClaw（如需AI助手功能）
npm install -g openclaw
```

#### 第二步：部署代理服务
```bash
# 创建目录
mkdir -p /opt/ber-report
cd /opt/ber-report

# 放置 proxy-server.js（见附件）

# 配置用户账号
# 编辑 proxy-server.js 中的 CONFIG.users 部分
```

```javascript
// proxy-server.js — 用户账号配置
const CONFIG = {
  port: 3000,
  erpHost: '8.130.178.145',   // 伯俊服务器IP
  erpPort: 90,
  erpPath: '/servlets/binserv/Rest',
  erpDb: 'kmyg',
  users: {
    // 每个客户一个账号
    'store_a': { appkey: 'xxx@xxx.com', appsecret: 'xxx', name: '某某店' },
    'store_b': { appkey: 'yyy@yyy.com', appsecret: 'yyy', name: '某某店' }
  }
};
```

#### 第三步：配置开机自启
```bash
# 使用 pm2 管理进程
npm install -g pm2
pm2 start /opt/ber-report/proxy-server.js --name ber-report
pm2 save
pm2 startup
```

#### 第四步：在伯俊ERP中配置菜单
1. 登录伯俊ERP → 开发平台 → 动作定义
2. 新增动作：
   - 名称：经营分析报表
   - 动作类型：JavaScript
   - 脚本内容：
```javascript
// 当前用户的 token（伯俊用户ID或其他标识）
pc.navigate('http://MacMini的IP:3000/report.html?token=当前用户', 'ifr', '经营分析报表')
```
3. 将该动作挂到菜单上

#### 第五步：部署报表页面

报表页面放在 Mac Mini 上，或者继续用 GitHub Pages。推荐放在 Mac Mini 本地。

```bash
# 报表页面放在 /opt/ber-report/ 下
# report.html → 报表入口页面
# 页面从 URL 取 token → 调代理服务 → 展示数据
```

### 3.3 客户自己选择AI模型（可选）
客户如果要用 AI 查询助手：
1. 注册 DeepSeek / OpenAI / 通义千问 等
2. 拿到 API Key
3. 你在小龙虾配置里填写客户的 Key
4. 客户即可在飞书/企业微信中通过对话查数据

## 五、你的护城河

### 客户自己搞不定的地方

| 环节 | 为什么客户搞不定 | 你的价值 |
|------|-----------------|---------|
| **字段映射** | 伯俊REST API返回的是数字数组，没有字段名 | 你维护了字段映射表 |
| **数据清洗** | 脏数据、空值、格式不一致 | 你做了校验处理 |
| **报表设计** | 客户不懂数据可视化 | 你提供专业报表模板 |
| **代理服务** | 客户不知道有中间层这回事 | 你控制了数据出口 |
| **权限整合** | 伯俊权限 + 报表权限的映射 | 你做了用户鉴权 |
| **持续更新** | 伯俊升级、接口变化 | 你负责适配 |

### 如何防止客户甩开你

**关键策略：代理服务是你的锁**

即使客户学会了小龙虾，他也没有字段映射表。没有这个表，REST API 返回的 `[35, 20230629, 10, 2500, 5]` 对小龙虾来说就是一堆没意义的数字。

你能做的就是：
1. **不透露字段映射关系**
2. **代理服务是闭源的**（编译或混淆）
3. **用户账号配置由你维护**
4. **年度维护 = 继续提供服务**
